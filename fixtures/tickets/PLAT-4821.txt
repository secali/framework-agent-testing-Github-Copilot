## Problem Statement

Since the deployment of v2.4.1 on May 24th, users with datasets larger than 
50,000 rows are experiencing 504 Gateway Timeout errors when generating reports 
through the `/api/v2/reports/generate` endpoint.

The Reports Service processes data synchronously, and for large datasets this 
exceeds the API Gateway's 30-second timeout threshold.

## Impact

- **Users affected**: ~340 enterprise accounts (tier 1 and tier 2)
- **SLA breach**: Report generation was guaranteed at < 25s for datasets up to 
  100,000 rows. We are now breaching this for anything > 50,000 rows.
- **Revenue at risk**: 3 enterprise clients have opened escalations.
- **Error rate**: 18% of all `/api/v2/reports/generate` requests fail (up from 0.2%)

## Steps to Reproduce

1. Authenticate as an enterprise user with a dataset > 50,000 rows
2. POST to `/api/v2/reports/generate` with payload:
   ```json
   {
     "dataset_id": "ds_enterprise_50k_plus",
     "report_type": "full_analysis",
     "include_charts": true,
     "date_range": { "start": "2026-01-01", "end": "2026-05-29" }
   }
   ```
3. Observe 504 error after 30 seconds

## Root Cause (Initial Analysis)

The v2.4.1 release changed the report generation from streaming to batch mode
(commit `a3f92c1`) to fix a chart rendering bug (PLAT-4756). This change 
inadvertently removed the incremental response mechanism.
